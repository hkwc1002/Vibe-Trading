# Low Absorb 生产化与外部联调简要说明

本任务是 A 类高风险生产化主线，目标是把 Low Absorb 从本地示例能力推进到真实数据可接入、异常可降级、外部联调可控制、Windows 本机可长期运行的状态。
范围包括：真实行情多源稳定性、英伟达 AI 服务器成本链半自动采集与人工审核、日线级真实回测、飞书全类型通知受控真实联调、本机部署与生产配置。
本阶段不处理页面中文化、首页/菜单/按钮文字、视觉体验或 UI 重构；这些需求后续独立立项。
A 股数据源可参考 `a-stock-data` skill，但必须封装在后端 provider 内，不能散落到前端或策略调用点。
真实外部副作用默认关闭；飞书真实发送必须由环境变量或受控配置显式开启。
如果实现需要新增依赖、修改项目配置、扩大前端范围或触碰金融安全边界，Claude Code 必须停止并请求 Codex 更新契约。
全程禁止真实券商下单、自动买卖、自动委托、券商交易执行 API 和一键交易。
实现完成后必须产出 `delivery.yaml`，经 Codex 审查生成 `review.yaml` 且结论为 `APPROVED_FOR_COMMIT` 后，才可在用户授权下进入版本阶段。
